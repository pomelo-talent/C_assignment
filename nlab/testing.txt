The testing strategy:

1. First, we can type "make all" in the terminal to generate "parse", "interp", "parse_s", "interp_s".


2. Next, we can type "make testparse" or "make testinterp" to check the outputs of many different .nlb files on the screen.


3. In addition, we can also type "make savetestinterp" which will save the interpreted outputs of many differnt.nlb files 
as .rst files respectively.


4. If we have typed "make savetestinterp" before, we can type "make autotest". The "autotest" will allow us to make comparisons 
between the interpreted output of one .nlb file (has been saved as a .rst file in step3) and our expected output of the .nlb file.
There is no output when we use autotest if the input file can be executed correctly by the parser and the interpreter.
Otherwise, a non-zero exit is made.

Usage of autotest:
     ./autotest <xxx.rst> <xxx.rst> 
      (the first xxx.rst file is the interpreted output of its homonymous .nlb file which is generated by "make savetestinterp")
      (the second xxx.rst file is our expected output of the xxx.nlb and we don't know whether the interpreted output of xxx.nlb 
      will match it now) 
      e.g. ./autotest ./expectedrst/setprinta.rst setprinta.rst
             ./autotest ./expectedrst/setprintc.rst setprintc.rst
